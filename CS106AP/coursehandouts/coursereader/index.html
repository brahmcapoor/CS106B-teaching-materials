The two most fundamental building blocks in any computer program are variables, which can store information, and functions which can perform actions.

Variables

>Defining and Assigning Variables

We use variables to store information about our program. Variables in computer science are similar, but not quite the same as variables you may have encountered in math. Like variables from math, variables in computer science act as placeholders for values with names we assign like x or y. Unlike variables from math, you don't set up equations and "solve for" their values. Instead, we as programmers assign values to our variables. In general, we write that

variable_name = some value

If, for instance, we write
the_answer = 42

then we are assigning the value 42 to the variable called the_answer. The first time that we assign a value to variable, we say that we are defining the variable. This distinction is important because we cannot use a variable before we have defined it. For instance, 
                        
>>> the_answer = 42 
>>> print(the_answer) 42 
>>> print(an_undefined_variable)
Traceback (most recent call last): File "<stdin>", line 1, in
<module> NameError: name 'an_undefined_variable' is not defined
Don't worry too much about the scary output; it translates to Python isn't a fan of undefined variables! There are many types of information that can be stored in variables For instance,
  • int variables store integers. Integers are numbers without decimal values like -81, 0, or 42. You might use an int to represent the number of students in a class.
  • float variables store numbers with decimal values. Floats are numbers like -9.876, 2.0, or 3.14159. Notice that 2.0, while here considered a float is also a valid integer value. We shall later discuss the implications of this, but for now we will work with the convention that 2 refers to the integer value while 2.0 refers to the float value. A float might be used to represent the average grade of students on a midterm.
  • bool variables, also known as Boolean variables, are a unique kind of variable that can only take one of two values, True and False. A bool could be used to represent whether or not a class is recorded (i.e. True means the class is recorded and False means it is not).
  • str variables (i.e. strings) store textual information. We'll be discussing strings in much greater detail in String Properties and Functions, but we'll give a brief description of them here as well. We surround strings using single quotation marks ('), so for instance, the sentence, You bet Bit's the best bot!, would be represented in code as 'You bet Bit's the best bot!'. We can also have what's called an empty string. This string represents a completely blank line of text and is written as two single quotes, ''. One simple example of a str variable would be one used to represent the name of a class!

Determining Variable Type
If you ever are unsure of what type a particular variable has, you can use the built-in type function to find out. For instance, 
                        
>>> woah_a_string = 'a cool string' 
>>> hey_a_float = 4.8 
>>> lit_an_int = 9
>>> yeet_a_bool = True
>>> print(type(woah_a_string)) 
#Prints: <class 'str'>
>>> print(type(hey_a_float)) 
#Prints: <class 'float'> 
>>> print(type(lit_an_int)) 
#Prints: <class 'int'>
>>> print(type(yeet_a_bool)) 
#Prints: <class 'bool'>

>Operations on Variables

Integers and Floats

There are a handful of predefined operations you can perform on numeric variables (e.g. int and float variables) that do roughly what you'd expect. These are operations like +, -, *, and /. Consider, 

                        
>>> a = 2.5 
>>> b = 4.0 
>>> c = a * b #c is now 10.0
>>> d = c + a #d is now 12.5 
>>> print(d) 12.5
There is one notable exception, namely the division of an int by another int. Consider, 
                            
>>> x = 5 
>>> y = 2 
>>> z = x / y 
>>> print(z) 
2.5
So, even though x is an int and y is an int, z is a float! There is another operation, // which refers to integer division. You can think of integer divison as regular division where the final result is rounded down to the nearest integer. So, 
                            
>>> x = 5 
>>> y = 2 
>>> w = x // y 
>>> print(w) 
2
This type of integer division will become an incredibly useful tool once we get to Image Algorithms. One common gotcha has to do with negative numbers. Consider -5 // 2. You might be tempted to say that the result is -2 since 5 // 2 is 2. However, since we always round down, -5 // 2 will actually be -3!

Updating a Variable
Sometimes we want to update the value stored in a variable. That means that we want to change the value it has stored, oftentimes in a way that relies on the value it currently has. For example, imagine that we have the grades of a few students in a class and we want to compute the class average. We could use the following code to do so: 
                                
>>> score_1 = 75.4
>>> score_2 = 89.7
>>> score_3 = 69.8
>>> avg_score = 0
>>> avg_score = avg_score + score_1 #avg_score is now 0 + 75.4 = 75.4
>>> avg_score = avg_score + score_2 #avg_score is now 75.4 + 89.7 = 165.1
>>> avg_score = avg_score + score_3 #avg_score is now 165.1 + 69.8 = 234.9
>>> avg_score = avg_score / 3 #avg_score is now 234.9 / 3 = 78.3
>>> print(avg_score)
78.3
This syntax of variable = variable +,-,*,/ value is so prevalent in programming that we have created an easy way to abbreviate it. Namely, variable +,-,*,/= value. The code we wrote above could be rewritten as 
                                
>>> score_1 = 75.4
>>> score_2 = 89.7
>>> score_3 = 69.8
>>> avg_score = 0
>>> avg_score += score_1
>>> avg_score += score_2
>>> avg_score += score_3
>>> avg_score /= 3
>>> print(avg_score)
78.3
This syntax is substantially cleaner and saves you the work of typing out some longer and more annoying lines.
Strings
The main "operation" that we can do with strings is known as string concatenation. String concatentation is the process of taking two strings and "gluing" them together. Concatention is done with the + operator so 'hello ' + 'world' would produce the string 'hello world'. Notice the space after the o in "hello". Without it, we would concatenate 'hello' + 'world' and produce 'helloworld', which may or may not be what we want given the context. As with integers and floats, we can use the += operator here. For example, 
                            
>>> s1 = '' #The empty string!
>>> s1 += 'I am'
>>> s1 += ' excited ' #notice the spaces here
>>> s1 += 'about CS 106AP!'
>>> print(s1)
'I am excited about CS 106AP!'
Booleans
Booleans have a unique set of operations called and, or, and not. If a and b are Booleans, a and b is True only if a and b are both individually True otherwise a and b is False. Now, a or b is True if at least one of a and b is True and False otherwise. Finally, we have not a. The operation not "flips" a Boolean so if a is True, not a is False and vice versa.

What makes Booleans really powerful is they are the language we use to tell if things are equal to, greater than, or less than one another. We've summarized how to do this in the following table.

Function Description
x == y True if x and y are equal, False otherwise.
x < y True if x is less than y, False otherwise.
x > y True if x is greater than y, False otherwise.
x <= y True if x is less than or equal to y, False otherwise.
x >= y True if x is greater than or equal to y, False otherwise.
A Common Mix Up
Notice that the way we test if x and y are equal is with == not = like you may be used to from math. This is a very common mistake to make! Always try and keep in mind that when you're assigning a value to a variable to use = and if you're testing equality to use ==.

Functions

>Defining a Function
Now that we have the ability to save information in our program using variables, we can use functions to actually do something with that data. We've already been using a function quite extensively, the print function, which allows us to output information. The most basic template of a function looks as follows 
                        
def do_something_awesome(param1, param2):
    #the function body! do cool things here
    return something
We can understand this template piece by piece. The keyword def (short for "define") simply tells Python that what comes next is going to be the definition of a function. What follows next is the name of the function (in this case we chose do_something_awesome, but in general you should choose a name that describes what the function you're writing actually does). In parentheses are the parameters, which act as inputs into the function. After the colon comes the function body,where you'll write the function will actually execute. At the very end of the function is a return statement, which acts as the output of the function. Both parameters and return statements are optional, and are only included when your function has inputs or outputs respectively. We'll see exactly how this plays out in some examples.
>A Basic Function
Consider the following function: 
                      
def is_even(x):
    y = x // 2
    return 2 * y == x
The intent of this function is to return True if the input x is even and False otherwise. We accomplish this by taking advantage of the rounding down feature of integer multiplication. If x is 4, then x // 2 will be 2 and so 2 * y will be 4. Since 4 == 4 is True, is_even return (i.e. output) True! If x is 5, then x // 2 will be 2, meaning that 2 * y will be 4. Since 4 == 5 is False, is_even will return False! Feel free to try out some other values to prove to yourself why this code works.

The variable x is what we call a parameter. When we're writing our method (like is_even), we don't know exactly what values will be inputted as our parameters, so we need to make our function able to handle any reasonable set of possible inputs. We've already walked through the logic of how if x is either an even or an odd number we'll get the right result out. If we want to actually call our function (that is, use it) we could write the following code once it has been defined: 
                    
>>> print(is_even(5))
False
>>> x = -4
>>> print(is_even(x)) #Does our program work for negative numbers? Go back and check!
True
>Multiple Parameters
Sometimes we want to write functions that have multiple inputs. For instance, let's say that we want to write a function that calculates the y-value of a point on a line. Recall that for a line with slope m and y-intercept b, y = mx + b. In code, we might express this as 
                    
def calculate_y(m, x, b):
    y = m * x + b
    return y
Now, in our console, we might write 
                        
>>> print(calculate_y(1, 2, 3)) #1 * 2 + 3 = 5
5
>>> print(calculate_y(3, 2, 1)) #3 * 2 + 1 = 7
7
The order in which inputs are passed into the function must match the order in which they're written in the function definition. In the first function call, m is 1, x is 2, and b is 3. In the second these definitions are reversed, yielding two very different results.
